vEwe

vEwe is a JS library used to manage events through an easy to extend view object. With jQuery as its sole dependency it provides a lightweight alternative for those wishing to avoid a full fledged "framework".

  • AMD (asynchronous module definition) ready
  • Unlimited inheritance in 2 flavors
  • Light weight prototypical view objects
  • Events with all the power of jQuery's .on()

Examples

101 - The Basics

We're going to start with a simple view that handles toggling between 2 DIVs.

Setup the HTML.

<div class="stage">
	<figure class="tada">
		<img >
	</figure>
	<figure class="box">
		<img src="../images/box.png">
	</figure>
</div>

Setup the CSS.

.tada{
	display: none;
}

Include the JS dependencies.

<script type="text/javascript" src="http://code.jquery.com/jquery-2.0.2.js"></script>
<script type="text/javascript" src="../../js/view.js"></script>

Define a view prototype.

viewPrototype = {
	'events': [
		['click','.box','hideBox'],
		['click','.tada','showBox']
	],
	'hideBox': function(e){
		// Hide the box and ta-da!
		e.data.$el.find('.box').hide();
		e.data.$el.find('.tada').show();
	},
	'showBox': function(e){
		// Show the box and hide animal
		e.data.$el.find('.tada').hide();
		e.data.$el.find('.box').show();
	}
}

Create a view from the previously defined prototype.

var view = viewFactory.create(viewPrototype);

On DOM ready turn the view events on.

view.$(function(){
	view.on();
});

201 - Inheritance

Now let's say you want to make a new view that inherits the methods available in the view from 101.

Define a second prototype.

viewPrototypeExtended = {
	'events': [
		['view.on','tada'],
		['click','.box','hideBox'],
		['click','.tada','showBox']
	],
	'tada': function(e){
		e.data.$el.find('.tada img').attr('src','../images/duck.png');
	}
}

Pass both prototypes into the view factory's create method so that a view can be generated using both prototypes.

view = viewFactory.create(viewPrototype, viewPrototypeExtended);

202 - Inheritance & Merged Events

The basic inheritance from 201 is pretty spiffy, but how much cooler would it have been if you hadn't had to supply all those duplicate events to each prototype? Surprise!

Define a new prototype. Notice that we aren't bothering to include events from the parent prototype.

viewPrototypeExtended = {
	'events': [
		['view.on', 'addSpin'],
	],
	'addSpin': function(e){
		e.data.$el.find('.tada > img').addClass('spintastic');
	}
}

Using our trusty create method we now specify a special type of inheritance.

view = viewFactory.create('inheritAndMergeEvents', viewPrototype, viewPrototypeExtended);
